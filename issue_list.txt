# イシュー一覧 Ver1
イシュー一覧

1. ノード右クリックでコピー/貼り付け/削除
目的:
- 既存ノードを複製して再利用したい
- ノードを安全に消したい
- 作成と削除を右クリック操作に統一したい

価値:
- 編集スピードが上がる
- 作業手順が直感的になる
- memory.jsonやコード.jsonを同時更新するフックを1か所に集約できるので破綻しにくい

難易度:
- 中
- ContextMenuStrip等の土台とID採番ロジック(コード.jsonの「最後のID」)は既存なので追加実装中心

フェーズ:
- フェーズ1候補


2. ノード検索とジャンプ
目的:
- ノード数が増えても目的の処理にすぐ飛びたい
- レイヤーをまたいでも対象ノードの場所を特定したい

価値:
- 既存フローの保守とレビューが速くなる
- 大規模化しても迷わない
- 再利用しやすい

難易度:
- 低～中
- memory.jsonにはノード名/座標/レイヤーIDがまとまっているのでインデックス化しやすい

フェーズ:
- フェーズ1候補


3. Undo / Redo
目的:
- 誤操作から即復帰したい
- 大きなレイアウト変更を試せるようにしたい

価値:
- 作業スピードを落とさず安全に編集できる
- レイアウト崩壊を恐れず試行できる

難易度:
- 高
- 画面上のボタンとmemory.jsonを同時に巻き戻す履歴スタックを新規で設計する必要がある

フェーズ:
- フェーズ2候補


4. 変数パネル (変数の定義と現在値の可視化)
目的:
- どこでどの値がセットされてどこで使われるかを明示したい
- 未定義のまま参照している箇所を検出したい

価値:
- デバッグしやすくなる
- 事故的な上書きや未定義参照を早期に発見できる
- 他人に説明しやすい

難易度:
- 高
- 実行エンジンから現在の変数状態をUIへ渡す仕組みが必要になるため新規性が高い

フェーズ:
- フェーズ3候補
