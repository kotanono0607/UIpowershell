# なぜUIpowershellを選ぶのか

**WinActorに勝てる3つの理由**

---

## 1. 完全無料 vs 年間90万円

| 項目 | WinActor | UIpowershell |
|------|----------|--------------|
| ライセンス費用 | 年間90万円～ | **無料** |
| 追加ライセンス | 1台ごとに課金 | **無制限** |
| 実行環境 | WinActor必須 | **EXEで単体動作** |

---

## 2. AIでノードを無限に追加できる

### WinActorの場合
```
新機能が欲しい
    ↓
NTTに依頼
    ↓
開発費用（数十万円～）
    ↓
数ヶ月待ち
    ↓
やっと使える
```

### UIpowershellの場合
```
新機能が欲しい
    ↓
AIに「〇〇するノード作って」と依頼
    ↓
5分で完成
    ↓
すぐ使える
```

### ノード追加の仕組み

UIpowershellのノードは **JSONファイル1つ + PowerShellファイル1つ** で構成されています。

```
ノード追加 =
  1. ボタン設定.json にエントリ追加
  2. 00_code/{処理番号}.ps1 を作成
  3. 完了！
```

AIに依頼すれば、この2ファイルを自動生成してくれます。

**詳細**: [ノード作成ガイド](./ノード作成ガイド.md)

---

## 3. 作ったRPAをEXE化して配布できる

### これがUIpowershell最大の強み

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  UIpowershell でワークフロー作成                            │
│       ↓                                                     │
│  output.ps1（PowerShellスクリプト）                        │
│       ↓                                                     │
│  ps2exe で EXE化                                            │
│       ↓                                                     │
│  「誰でも動かせる単体EXE」として配布                        │
│                                                             │
│  → PowerShell不要                                           │
│  → WinActorライセンス不要                                   │
│  → ダブルクリックで実行                                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### WinActorとの決定的な差

| 観点 | WinActor | UIpowershell |
|------|----------|--------------|
| **実行に必要なもの** | WinActorライセンス | **EXEだけ** |
| **配布** | 不可（ライセンス必要） | **EXEを渡すだけ** |
| **相対パス** | 制限あり | **完全対応** |
| **実行環境** | WinActor必須 | **.NET Frameworkのみ** |
| **カスタマイズ** | 制限あり | **PowerShellで自由自在** |

### 相対パスが使える = ポータブル

```
📁 業務自動化ツール/
├── 処理ツール.exe      ← 生成したEXE
├── input/
│   └── データ.xlsx     ← 入力ファイル
├── output/
│   └── 結果.xlsx       ← 出力ファイル
└── config.json         ← 設定ファイル

→ このフォルダごとコピーすれば、どこでも動く
→ USBメモリで持ち運び可能
→ パスの書き換え不要
```

## ユースケース

### 1. 社内ツール配布

```
経理担当「毎月のExcel集計を自動化したい」

開発者:
1. UIpowershellでワークフロー作成（30分）
2. ps2exeでEXE化（1分）
3. 「集計ツール.exe」を共有フォルダに置く

経理担当:
- EXEをダブルクリック
- 同じフォルダのExcelを自動処理
- 完了！

結果:
- PowerShellの知識不要
- WinActorライセンス不要
- 保守コストほぼゼロ
```

### 2. クライアントへの納品

```
クライアント「業務を自動化してほしい」

開発者:
1. UIpowershellでRPA開発
2. EXE化して納品
3. マニュアル付き

クライアント:
- EXEを実行するだけ
- 追加ライセンス費用ゼロ
- 設定ファイルでカスタマイズ可能

結果:
- 納品後の追加費用なし
- 保守はEXE差し替えだけ
- クライアント満足度UP
```

### 3. 自社製品として販売

```
あなた:
1. 特定業務向けRPAを開発
2. EXE化してパッケージ化
3. 「〇〇業務自動化ツール」として販売

顧客:
- 買い切りで使える
- 月額費用なし
- カスタマイズ依頼も可能

結果:
- RPAを「製品」として販売可能
- WinActorでは絶対にできないビジネスモデル
```

## ps2exe の使い方（参考）

```powershell
# ps2exe のインストール
Install-Module -Name ps2exe -Scope CurrentUser

# EXE化
Invoke-ps2exe -InputFile "output.ps1" -OutputFile "処理ツール.exe" -NoConsole

# オプション
#   -NoConsole    : コンソール非表示
#   -RequireAdmin : 管理者権限要求
#   -Icon         : アイコン指定
```
